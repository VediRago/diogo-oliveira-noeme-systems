Building the Cosmology
One of the applied methods behind this repository's principles.

The Method
A cosmology should generate stories rather than explain them.

The rule underneath everything else: experience requires difference. A single identity separated across different states, worlds, and outcomes creates difference; difference allows change; change is what makes experience possible. Connection gives those separate experiences meaning — but complete unity removes the difference experience depends on, so when unity becomes too stable, fragmentation returns so experience can continue.

Explanation
The cosmology's job isn't to be explained to the reader. It's to be felt through symptoms, repeated patterns, failed explanations, and — eventually — Unity and the hospital ending. If a reader could summarize the whole system after one scene, the cosmology would have failed at its actual job: generating story pressure, not delivering lore.

The cycle itself: fragmentation enables experience, experience produces connection, connection trends toward unity, unity creates instability, instability triggers re-fragmentation.

Application
Building outward from that cycle exposed several questions about the same underlying structure. Each one examines the Continuity Cycle from a different perspective rather than extending it into a new layer:

**Who tracks the cycle?** Not a god, not a villain — something closer to a librarian with access to the whole library but no need to read every book at once. See [Qalm](../vault/building-the-cosmology/qalm.md).

**What actually experiences the cycle?** Separate lives, connected to one underlying self, who don't share memory — only residue: repeated motifs, familiar behaviors, recognition without explanation. See [Fragments](../vault/building-the-cosmology/fragments.md).

**What does instability look like from inside a life?** A recurring pattern that reads as biological or tragic long before it reads as structural. See [The Condition](../vault/building-the-cosmology/the-condition.md).

**How does a reader recognize any of this without being told?** The same perception shape repeats at every scale — partial perception, recognition, incomplete interpretation, deeper pattern — for characters, for the reader, and for Qalm itself. See [The Fractal Perception Pattern](../vault/building-the-cosmology/fractal-perception-pattern.md).

**What breaks the cycle?** A fragment the system can no longer classify. See [Kael as Anomaly](../vault/building-the-cosmology/kael-as-anomaly.md).

None of these questions were invented separately. Each one examines the same cycle from a different perspective — who watches it, who lives inside it, what it feels like, how it is recognized, and what happens when it fails to resolve correctly.
